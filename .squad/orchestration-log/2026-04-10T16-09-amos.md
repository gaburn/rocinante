## API Endpoints

1. **POST /api/sessions/archive** — Startup sync; replace server archive with client state
2. **POST /api/sessions/archive/add** — Idempotent add single session to archive
3. **POST /api/sessions/archive/remove** — Idempotent remove single session from archive
4. **GET /api/sessions/archive** — Read current archived IDs
5. **GET /api/sessions?includeArchived=[true|false]** — List sessions (default: exclude archived)
6. **GET /api/sessions/search** — Search all sessions; results include `isArchived: boolean` flag

## Technical Notes

- Archive mutations invalidate response cache; cache key includes `includeArchived` flag to prevent stale data
- Backward compat: If archive store uninitialized, GET returns all sessions (pre-Phase-3 behavior)
- Search endpoint always searches full session store; archive awareness delegated to frontend for rendering
